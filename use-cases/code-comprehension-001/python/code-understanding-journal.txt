Excercise Part 1: Understanding a Specific Feature
The task creation and status update feature in the TaskManager codebase connects the CLI, logic, and storage layers to manage task lifecycles.
When a user runs a command like python cli.py add "Finish report", the CLI parses input and calls TaskManager.create_task(), which validates dates, assigns priority and status enums, creates a Task object, and saves it through the storage layer.
The update_task_status() method retrieves a task by ID, converts the new status to an enum, and updates timestamps when marked as done.
This flow demonstrates clear separation of concerns and domain-driven design.
